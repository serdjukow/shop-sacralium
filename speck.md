Общие положения
Логотип на страницах оплаты - это ссылка на https://sacralium.game
Внизу каждой страницы за пределами скролла расположен футер с черным фоном, содержащий набор необходимых по законодательству реквизитов компании-оператора сервиса.
Страница оплаты располагается на shop.sacralium.ru 
Интеграция
Этап 1. Интегрируем страничку shop.sacralium,ru с мерчантом и игрой, смотрим прохождение платежей. В игре [ + ] ведет как и ранее на XSolla.

Этап 2. В игре при клике на [ + ] проверяем по гео-IP. если клиент из РФ - то переадресация на shop.sacralium.ru, если не из РФ - как сейчас в XSolla.
Поиск пользователя

Если в URL не передан параметр get с ID персонажа ?charID=2134, то форма отображается с дизейблд кнопками и полем ввода для имени или ID персонажа.
Screenshot 1.png

Если поле заполнено (>=1 символа) - кнопка в поле ввода становится красной, при OnHover такая кнопка становится чуть более насыщенно красной.
Screenshot 2.png

После нажатия на кнопку происходит запрос к серверу на поиск персонажа, если введен корректный ID или имя - то переадресация на страницу с get-параметром, в котором указан ID персонажа.  Если параметр указан, но ID некорректный, происходит переадресация на страницу без указанного параметра (с полем ввода).
Выбор предмета покупки

При корректном ID персонажа, в правом верхнем углу отображается фраза “Покупка для персонажа Имя”, а кнопки становятся энейблд.
Screenshot 3.png

OnHover состояния фона плашки и кнопки. При наведении в любое место плашки (в т.ч. на кнопку) плавно за 0.2 сек меняется ее фон. При наведении на непосредственно кнопку - фон кнопки.
Screenshot 4.png

При клике на кнопку покупки, происходит переадресация на страницу мерчанта, в том же окне, с выбором варианта платежа. Предположительно, это будет ЮКасса.

В случае, если гео-IP показывает, что клиент не из РФ, то переадресация на страницу выбора варианта платежа XSolla, с сохранением выбранного ID персонажа и ID покупки. Если это прям сложно сделать - то не делаем.

Статус покупки

После совершения покупки - при возврате в магазин - отображаем статус покупки. В гет-параметрах - ID транзакции.
Screenshot 5.png

Если деньги не пришли - то показываем вот такую страницу и чекаем статус раз в 10 секунд (авто обновление страницы).
